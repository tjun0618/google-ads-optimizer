# Google Ads Optimization Skill — Amazon Affiliate Edition

> 版本: v3.0 | 2026-04-28
> 定位: 专为YP平台Amazon联盟营销客设计的广告优化技能
> 上游依赖: google-ads-v6.3（广告创建）→ google-ads-diagnosis（诊断）→ **本技能（优化）** → 持续迭代
> 核心变化: v3.0起，诊断由诊断技能独立完成，本技能消费诊断结果，聚焦于「制定优化方案 + 生成清洁版广告」

---

## v3.0 架构变革：诊断与优化分离

### 为什么分离？

v2.2中，优化技能同时承担诊断和优化两个职责，导致：
1. 职责不清——诊断是"发现问题"，优化是"解决问题"，混在一起
2. 重复劳动——如果用户已经跑了诊断技能，优化技能又要重新做一遍
3. 流程冗长——从接收文件到最终清洁版，9个Step太长

### 新架构

```
google-ads-v6.3（创建技能 v6.4.0）
    ↓  生成初始清洁版
  投放到 Google Ads
    ↓  收集投放数据（Excel报告）
google-ads-diagnosis（诊断技能）
    ↓  输出诊断报告（P0/P1/P2 + 利润模型 + 否定词清单）
    ↓
google-ads-optimizer v3.0（本技能）★ 你在这里
    ↓  消费诊断报告 → 制定优化方案 → 生成清洁版广告
    ↓
  回到 Google Ads 执行优化
    ↓  收集新数据 → 再次诊断 → 再次优化（循环）
```

### 技能间数据传递

| 数据项 | 诊断技能产出 | 本技能消费 |
|--------|-------------|-----------|
| 利润模型 | 盈亏平衡CPC、安全上限、利润评级 | 作为优化决策的核心约束 |
| P0/P1/P2问题 | 分级问题清单 + 数据证据 | 转化为具体优化行动 |
| 否定词清单 | Account/Campaign/AG三级否定词 | 直接写入清洁版广告文件 |
| 优化建议 | 立即/本周/观察期三级行动 | 作为优化方案的基础框架 |
| 止损框架 | 止损线计算结果 | 写入清洁版广告文件头部注释 |
| 搜索字词分析 | 意图分布、旅程断点 | 驱动文案调整和AG启动优先级 |

## 核心差异：为什么Affiliate优化不同？

作为Amazon Affiliate，你的优化约束与普通电商完全不同：

| 约束 | 普通电商 | Amazon Affiliate (YP平台) |
|------|---------|--------------------------|
| 落地页 | 可控，可A/B测试 | 不可控，直接到Amazon |
| 转化追踪 | Google Ads像素完整追踪 | 无法追踪，只能通过YP平台看品牌级转化 |
| 数据粒度 | 关键词/搜索词级归因 | **品牌级聚合**，无法追踪到具体搜索词 |
| 出价策略 | Target CPA / Maximize Conversions 可用 | 只能用 Maximize Clicks 或人工出价 |
| 受众 | 可用RLSA再营销 | 无网站，无法收集受众 |
| 利润模型 | 销售收入 - 广告成本 | YP平台佣金 - Google Ads支出 |
| Cookie窗口 | 无限制（自己的转化） | 24小时Amazon Associates Cookie |

**Affiliate优化的核心公式**：

```
盈亏平衡CPC = 客单价 × 佣金率 × Amazon预估转化率
安全CPC上限 = 盈亏平衡CPC × 安全系数(0.3-0.7)
```

所有优化动作必须围绕**降低无效点击成本**和**提高流量购买意图**展开。

## Step 0: 接收并验证文件

### 输入
- 用户提供的文件路径列表

### 执行
1. 确认诊断报告文件存在且可读
2. 确认原广告方案文件（清洁版）存在且可读
3. 如有Excel报告，一并确认
4. 从诊断报告中提取核心信息：品牌名、产品名、ASIN、价格、佣金率、盈亏平衡CPC
5. 从广告方案中提取产品信息，与诊断报告交叉验证

### 边界条件处理

**缺少诊断报告**:
```
如果用户未提供诊断报告:
  1. 告知用户优化技能需要先完成诊断
  2. 列出诊断技能的触发词（广告诊断、投放诊断、ads diagnosis等）
  3. 询问用户是否要跳过诊断直接优化（进入直接优化模式）
```

**直接优化模式（Fallback）**:
```
如果用户选择跳过诊断:
  1. 要求用户提供: 广告方案文件 + 搜索关键字报告 + 搜索字词报告
  2. 本技能临时执行轻量诊断（复用诊断技能的核心检测逻辑）:
     - 利润模型计算
     - P0-1 广泛匹配失控检测
     - P0-2 广告结构检测
     - P0-3 零否定操作检测
  3. 但仅检测P0级问题，跳过P1/P2
  4. 在报告中标注"轻量模式，建议补充完整诊断"
```

**文件不存在或不可读**:
```
如果任一必要文件不存在:
  1. 列出所有缺失的文件路径
  2. 告知用户需要提供的文件
  3. 停止执行，等待用户重新提供
```

**产品信息不匹配**:
```
如果诊断报告和广告方案中的产品信息不一致:
  1. 列出不一致的字段（ASIN、价格、佣金率等）
  2. 告知用户并要求确认
  3. 优先以诊断报告为准（因为诊断基于实际投放数据）
```

## Step 1: 解析诊断报告，提取诊断结论

### 执行
从诊断报告中提取以下关键信息（具体数据结构见诊断报告的"投放数据总览"和"问题诊断"章节）：

**必须提取的信息**：
1. **利润模型**：盈亏平衡CPC、安全上限、利润评级、预估佣金
2. **投放数据**：总费用、总点击、总展示、平均CPC、CTR
3. **问题清单**：P0/P1/P2问题列表（每个问题含数据证据）
4. **否定词**：Account/Campaign/AG三级否定词清单
5. **优化建议**：立即/本周/观察期三级行动
6. **止损框架**：止损线计算结果
7. **搜索词意图分析**：意图分布、旅程断点、意图-广告错配（如有）

### 输出
- 诊断结论摘要（向用户展示已成功提取的关键信息）
- 如诊断报告缺少上述任一信息，提示用户诊断报告不完整，建议补充运行诊断技能

## Step 6: 制定优化方案

基于诊断报告的结论，制定具体可执行的优化方案。

### 6.1-6.2 诊断问题→优化行动映射

> 详见 → `references/optimization-analyzer.md` 中的「映射总表」

对诊断报告中的每个问题（P0/P1/P2），按映射总表转化为具体优化行动。核心映射逻辑：
- **P0（必须立即修复）**：广泛匹配收紧、结构重建、否定词批量添加、竞品词排除
- **P1（本周修复）**：出价调整、零效果词暂停、非购买意图词否定、预算重分配、文案合规修正
- **P2（观察优化）**：匹配类型渐进收紧、AG拆分重组、意图标注补全

### 6.3 否定关键词调整

直接采用诊断报告的三级否定词清单，合并到原广告方案中：

**合并规则**:
- 诊断报告中新增的否定词 → 追加到对应级别
- 原广告方案中已有的否定词 → 保留
- 两者重复的 → 去重

### 6.4 关键词调整（基于利润分析）

详见 → `references/optimization-analyzer.md` 中的关键词调整规则。

核心原则：
- **亏损词**：CPC > 盈亏平衡CPC → 暂停（在清洁版中用 `# 暂停: [原因]` 注释）
- **高利润词**：CPC < 安全上限 × 0.5 → 加码（提高出价10-20%）
- **广泛匹配失控**：无关触发≥5条 → 收紧匹配类型
- **零效果词**：展示>200且点击=0 → 暂停

### 6.5 出价策略调整（Affiliate专用）

详见 → `references/optimization-analyzer.md` 中的出价策略规则。

核心原则（所有Campaign一律）：
- **策略**：Maximize Clicks（Affiliate无法追踪转化）
- **Brand Campaign上限**：盈亏平衡CPC × 0.7
- **Category Campaign上限**：盈亏平衡CPC × 0.6
- **Pain Points Campaign上限**：盈亏平衡CPC × 0.5

### 6.6 文案调整原则

> 文案调整必须遵循 google-ads-v6.3 创建技能（v6.4.0）的所有规则：
> - 标题截断铁律（≤28字符）
> - 引号包裹铁律
> - 品牌名铁律
> - EPC-A1~A6 联盟专用规则
> - EPC-18~EPC-30 系列规则（YMYL合规、语义重叠检测等）
> - QA-14~QA-26 全部检查点

核心调整方向（基于诊断发现的意图错配）：
- **调研意图**（reviews/reddit）→ 社会证明型文案（评分、评论数）
- **探索意图**（best/top）→ 差异化卖点型文案
- **痛点意图**（tired of/fix）→ 痛点共鸣型文案
- **Affiliate注意**：CTA必须导向Amazon，文案与产品页一致

### 6.7 预算分配建议

基于诊断报告的Campaign分布数据和利润分析：

核心原则：
- Category Campaign利润效率通常最高 → 优先启动和分配预算
- Brand Campaign CPC较高 → 有订单支撑时维持，无订单时降预算
- Pain Points Campaign转化链路长 → 保守预算

### 6.8 政策合规与库存检查

在生成具体优化方案前，执行Affiliate专用合规检查（详见 → `references/amazon-affiliate-checklist.md`）：

**YP平台与Amazon合规**:
- [ ] 广告文案中的价格是否与Amazon一致（或标注"Price may vary"）
- [ ] 是否存在 incentivize 点击的措辞
- [ ] YP联盟链接是否正确

**Google Ads政策合规**:
- [ ] 落地页是否为Amazon实际产品页（非Bridge Page）
- [ ] 广告文案是否存在夸大声明
- [ ] 是否存在虚假折扣/优惠声明

**库存与价格风险**:
- [ ] Amazon产品当前是否可用
- [ ] 价格是否有大幅波动（>20%）
- [ ] 评分是否有下降（<4.0或近期大量负面评价）

### 清洁版文件生成规则

1. **基于原广告方案修改**（不是从头创建）：
   - 读取原清洁版文件作为基础
   - 应用Step 6中的所有调整
   - 保留未调整的部分不变

2. **版本号递增**：
   - 原版本: v1.0 → 优化后: v1.1
   - 原版本: v6.4.2 → 优化后: v6.4.3
   - 重大结构调整: v1.0 → v2.0

3. **格式要求**（与google-ads-v6.3的Ready-To-Use格式完全一致）：
   - 关键词标注匹配类型（精确=[kw]、词组="kw"、广泛=+kw）
   - 标题≤30字符（标注字符数）
   - 描述≤90字符（标注字符数）
   - 否定关键词用 - 前缀
   - 包含完整广告扩展（Sitelinks/Callouts/Structured Snippets）
   - 包含出价策略和预算
   - 包含Amazon联盟链接注释

4. **优化变更注释**（让用户清楚看到改了什么）：
   - 暂停的关键词：`# ⛔ 暂停: [原因] | 亏损¥[X]`
   - 新增的关键词：`# ✅ 新增 | 利润评级: [X] | 来源: [搜索词]`
   - 修改匹配类型：`# 🔄 改匹配: [原]→[新] | 原因: [原因]`
   - 修改的标题：`# 🔄 修改: [原标题]→[新标题] | 原因: [原因]`

5. **Affiliate专用注释**：
   - 文件头部包含利润模型参数
   - 每个关键词后标注利润评级和目标意图
   - 止损线注释

### 清洁版文件结构

> 详见 → `references/report-generator.md` 第3节「清洁版广告文件格式」

文件结构遵循 google-ads-v6.3 的 Ready-To-Use 格式，新增以下 Affiliate 专用内容：
- **文件头部注释**：利润模型参数、止损线、诊断报告来源
- **变更注释系统**：⛔暂停 / ✅新增 / 🔄改匹配 / 📉降出价（详见 optimization-analyzer.md）
- **关键词标注**：利润评级 + 目标意图

## Step 8: 输出优化摘要报告

> 详见 → `references/report-generator.md`

### 报告结构

优化摘要报告是诊断报告的**下游补充文档**，不重复诊断内容，聚焦于"做了什么优化"：

1. **优化概述**（1-2句话：基于X诊断报告，执行了Y项优化）
2. **优化行动清单**（具体做了什么，可对照诊断P0/P1/P2）
3. **文件变更对照表**（原文件 vs 新文件的关键差异）
4. **优先级执行清单**（立即/本周/观察期）
5. **预期效果与风险提示**
6. **下一步建议**（何时再次诊断/优化的时间表）

### 输出文件

| # | 文件 | 路径 | 说明 |
|---|------|------|------|
| 1 | 优化摘要报告 | `brands/[BrandName]/[BrandName]-[ProductSlug]-Optimization-Summary-v[X].md` | 优化行动记录（补充诊断报告） |
| 2 | 清洁版广告 | `brands/[BrandName]/[BrandName]-[ProductSlug]-Ready-To-Use-v[X].[Y].md` | 可直接执行的优化后广告 |

> 注意：v3.0起不再单独生成完整的优化报告（Optimization-Report），因为诊断报告已经涵盖了分析内容。优化技能输出的是**优化摘要**（记录做了什么）+ **清洁版广告**（具体怎么改）。
