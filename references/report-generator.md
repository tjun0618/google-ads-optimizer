# Report Generator Reference — Google Ads Optimization Skill v3.0

> 本文件定义优化技能 v3.0 的输出规范。
>
> **v3.0架构**：诊断与优化分离。
> - 诊断技能产出：诊断报告（包含完整分析）
> - 本技能产出：优化摘要报告（记录"做了什么优化"）+ 清洁版广告文件（可直接执行）
>
> **重要**：本技能不再输出完整的优化分析报告（Optimization-Report）。
> 所有分析内容（利润模型、P0/P1/P2诊断、用户意图分析等）由诊断技能完成。
> 本技能只输出"基于诊断结果做了什么优化"的摘要和可执行的清洁版广告。

---

| 2 | 清洁版广告 | `[Brand]-[ProductSlug]-Ready-To-Use-v[X].[Y].md` | 可直接导入Google Ads Editor的广告文件 |

## 3. 清洁版广告文件格式

### 文件命名

```
[品牌名]-[产品Slug]-Ready-To-Use-v[X].[Y].md
```

版本号规则:
- 原版本: v1.0 → 第一次优化后: v1.1 → 第二次优化后: v1.2
- 重大结构调整: v2.0

### 文件结构

与 google-ads-v6.3 的 Ready-To-Use 格式完全一致，但新增 Affiliate 专用注释和变更注释：

```markdown
# [品牌名] [产品名] — Ready To Use v[X].[Y]（优化版）
# 优化基于: [诊断报告文件名]
# 创建技能版本: google-ads-v6.3 v6.4.0
# 产品: [产品名] | Price: $[价格] | Commission: [X]% | Breakeven CPC: $[X]
# ASIN: [ASIN] | Safe CPC Cap: $[X]
# 止损线: [X]次点击 / ¥[X] / [X]天

---

# NEGATIVE KEYWORDS

## Account-Level
# --- 原有保留 ---
-[原有否定词1], ...
# --- ✅ 新增（基于诊断报告） ---
-[新否定词1], ...

## Campaign 1 (Brand) Negative
-[否定词], ...

---

# CAMPAIGN 1: BRAND TERMS

## Ad Group 1A: Brand Core

### Keywords
1. [keyword1]  # 利润评级: A | 目标意图: Brand
2. [keyword2]  # ⛔ 暂停: CPC超盈亏平衡 | 亏损¥[X]
3. "keyword3"  # 🔄 改匹配: +kw→"kw" | 原因: 广泛匹配触发无关词
4. [keyword4]  # ✅ 新增 | 利润评级: A | 来源: 搜索词报告

### Headlines
1. [标题1] (XX chars) — 意图: Brand
2. [标题2] (XX chars) — 🔄 修改: 原"Stop Frizz"→现"Targets Frizz" | P1-9文案合规

### Descriptions
1. [描述1] (XX chars) — 意图: Brand

---

# AD EXTENSIONS
...

# BIDDING STRATEGY
- Campaign 1 (Brand): Maximize Clicks | Max CPC: $[X] | $[预算]/day
- Campaign 2 (Category): Maximize Clicks | Max CPC: $[X] | $[预算]/day
# Affiliate专用: 所有Campaign使用Maximize Clicks

# AFFILIATE LINK
[Amazon Associates联盟链接]
```

### 格式规范

1. **关键词匹配类型标注**: 精确=[kw] / 词组="kw" / 广泛=+kw
2. **否定关键词**: 用 `-` 前缀
3. **字符限制**: 标题≤30字符、描述≤90字符，每条标注字符数
4. **Affiliate专用注释**: 利润评级、目标意图、止损线、联盟链接
5. **变更注释**: ⛔暂停 / ✅新增 / 🔄改匹配 / 📉降出价

## 4. 报告风格指南

### 语言
- 中文为主，英文术语保留原文（Campaign、Ad Group、CTR、CPC、EPC等）
- 货币同时标注CNY和USD（如 ¥556.44 (~$76)）
- 利润评级用中文标注（A-高利润、B-盈利、C-边缘、D-亏损）

### 数据呈现
- 使用表格呈现结构化数据
- 关键数字加粗
- 百分比保留2位小数

### 行动项
- 今天必须做的用 `- [ ]` 清单格式
- 每个行动项具体、可执行
- Affiliate专用行动项标注利润影响（如"预计节省¥X/天"）
