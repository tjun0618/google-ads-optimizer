# Google Ads Optimization Skill

> 版本: v1.0 | 2026-04-25
> 定位: 广告创建技能(google-ads-v6.3)的下游配套技能，形成"创建→投放→数据反馈→优化"闭环

---

## 技能定位

本技能在广告创建技能(google-ads-v6.3)生成广告并投放后，基于Google Ads后台导出的实际投放数据进行数据驱动优化。核心流程：

```
用户上传3个文件 → 数据解析 → 诊断分析 → 优化建议 → 调整后方案
   (广告方案)    (Excel报告)   (问题识别)   (具体行动)   (可直接执行)
```

---

## 输入要求

用户需提供以下3个文件：

| # | 文件 | 格式 | 来源 |
|---|------|------|------|
| 1 | 广告方案文件 | .md | 之前由google-ads-v6.3生成的Ready-To-Use清洁版 |
| 2 | 搜索关键字报告 | .xlsx | Google Ads后台导出 → 报告 → 搜索关键字 |
| 3 | 搜索字词报告 | .xlsx | Google Ads后台导出 → 报告 → 搜索字词 |

**文件命名约定**：建议将3个文件放在同一品牌目录下，如 `brands/[BrandName]/`

```
Step 0: 接收文件并验证匹配性
  ↓
Step 1: 解析Excel报告（references/data-parser.md）

Step 2: 产品-报告匹配性验证

```

## Step 0: 接收文件并验证匹配性

### 输入
- 用户提供的3个文件路径

### 执行
1. 确认3个文件都存在且可读
2. 读取广告方案文件中的产品名称、ASIN、价格
3. 读取Excel报告中的关键词/字词样本，判断是否与广告方案的产品匹配

### 匹配性判断规则
- 搜索关键字报告中的关键词应与广告方案中的产品品类一致
- 例如：广告方案是吹风机(hair dryer)，报告中应包含hair dryer相关词
- 如果完全不匹配（如广告是喂猫机但报告全是猫砂盆词），需立即告知用户并确认

### 输出
- 产品信息确认摘要
- 匹配性判断结果（匹配/不匹配/需确认）

### 边界条件处理

**文件不存在或不可读**:
```
如果任一文件路径不存在:
  1. 列出所有缺失的文件路径
  2. 告知用户需要提供的文件
  3. 停止执行，等待用户重新提供
```

**数据量不足（<10条有效记录）**:
```
如果搜索关键字报告的有效关键词<10个:
  1. 告知用户数据量可能不足以进行有效分析
  2. 建议延长投放时间或增加预算以积累更多数据
  3. 继续执行分析，但在报告中标注"数据量有限，建议谨慎采纳"
```

**两个报告时间范围不一致**:
```
如果搜索关键字报告和搜索字词报告的时间范围不同:
  1. 提取两个报告的时间范围
  2. 告知用户时间范围不一致可能影响分析准确性
  3. 建议重新导出同一时间范围的报告
  4. 继续执行分析，但在报告中标注时间范围差异
```

## Step 3: 用户意图分析

详见 → `references/user-intent-analyzer.md`

### 核心输出
- 搜索字词按意图层级分类（购买决策/产品调研/方案探索/问题认知/信息浏览）
- 意图分布统计（各层级的字词数、点击、费用、转化）
- 典型用户旅程映射（从搜索词还原用户的决策路径）
- 旅程断点识别（哪些阶段无广告覆盖）
- 意图-广告匹配度检查（当前广告是否回应了用户的真实意图）
- 基于意图的文案优化建议

### 意图分析如何驱动优化

**发现1: 意图分布失衡**
- 如果90%点击来自"产品调研"意图 → 说明用户都在做功课，文案应强化社会证明
- 如果"方案探索"意图完全缺失 → 说明Category Campaign未启动，漏掉了最大流量池

**发现2: 意图-广告错配**
- 用户搜"reviews"但看到硬卖广告 → 调整文案为调研导向
- 用户搜"best"但看到品牌广告 → 启动Category Campaign

**发现3: 用户旅程断点**
- 发现用户从"tired of heavy dryer"开始 → 但无Pain Points Campaign → 新增否定词和启动Campaign

## Step 1: 解析Excel报告

详见 → `references/data-parser.md`

### 核心输出
- 搜索关键字报告：关键词列表、匹配类型、广告组、展示/点击/费用/转化
- 搜索字词报告：字词列表、匹配类型、广告组、已添加/已排除状态、展示/点击/费用/转化
- 关键汇总指标：总费用、总点击、总展示、总转化、整体CTR、整体CPC、整体CPA

## Step 2: 产品-报告匹配性验证

### 执行
1. 从搜索字词报告中提取出现频率最高的核心词（排除停用词后）
2. 与广告方案文件中的产品品类关键词对比
3. 如果搜索字词与广告方案的品类完全不同，标记为"产品错配"

### 产品错配处理
- 告知用户：报告中数据对应的是另一个产品
- 查找该产品对应的广告方案文件
- 经用户确认后，使用正确的广告方案文件继续优化

## Step 4: 诊断分析

详见 → `references/optimization-analyzer.md`

### 核心输出

- 每个问题附带数据证据和优化建议
- 关键词调整清单（暂停/修改匹配类型/调整出价）
- 否定词新增清单

## Step 5: 生成优化方案

> **用户确认检查点**: 在生成最终优化方案前，向用户展示诊断发现的核心问题摘要（P0/P1问题列表），确认优化方向后再生成详细方案。

### 展示给用户的核心问题摘要模板

```markdown
## 诊断发现摘要

### P0 致命问题（必须处理）
1. [问题1] — 影响:[描述]
2. [问题2] — 影响:[描述]

### P1 严重问题（建议处理）
1. [问题1] — 影响:[描述]
2. [问题2] — 影响:[描述]

### 优化方向
- 预计暂停 [N] 个关键词
- 预计添加 [N] 个否定词
- 预计启动 [N] 个未运行的广告组
- 预计调整 [N] 个关键词的出价

**请确认以上优化方向后，我将生成详细的优化报告和调整后的广告文件。**
```

### 4.1 否定关键词调整

根据搜索字词报告，生成新增否定词列表：

**账户级否定词新增规则**：
- 竞品品牌名（搜索字词中出现但非己方品牌的品牌词）
- 完全无关品类词（如广告是吹风机但触发了bonnet hair dryer、diffuser等）
- 不同产品类型词（如reverse hair dryer、stick blow dryer）
- 非目标语言词（如西班牙语、法语等美国市场非英语搜索）
- 教程/信息类搜索（tutorial、how to、without rollers等）
- 极低意图泛词（amazon hair dryer、hair dryer heat等）

**Campaign级否定词新增规则**：
- Brand Campaign：排除社区讨论平台（-reddit、-quora、-forum）、排除品牌旗下其他产品线
- Category Campaign：排除二手/维修/配件类
- Pain Points Campaign：保留竞品对比词（vs、compare、alternative），这些在痛点Campaign中有价值

### 4.2 关键词调整

| 调整类型 | 判定条件 | 操作 |
|----------|---------|------|
| 暂停 | 展示>0且点击=0，或已暂停且无历史价值 | 暂停 |
| 收紧匹配 | 广泛匹配触发大量无关词（搜索字词报告中≥5条无关触发） | 改为词组或精确 |
| 降低出价 | CPC > 整体平均CPC的1.5倍 且 0转化 | -15%~-30% |
| 提高出价 | CTR > 整体CTR的2倍 且 有转化 或 费用效率高 | +10%~+20% |
| 新增关键词 | 搜索字词报告中"已添加"但未作为关键词添加的高价值词 | 添加 |

### 4.3 出价策略调整

| 场景 | 当前策略 | 建议调整 |
|------|---------|---------|
| Brand Campaign 0转化 | Maximize Conversions | 改为 Maximize Clicks（先积累数据） |
| Category Campaign 未启动 | — | 新建，Maximize Conversions |
| Pain Points Campaign 未启动 | — | 新建，Maximize Conversions |
| 某AG CPA > 盈亏平衡CPA的150% | Maximize Conversions | 降低Target CPA 20%或暂停 |

### 4.4 文案调整原则

**Brand Campaign 文案**：
- 如果搜索字词中大量reviews/reddit相关词 → 用户处于调研阶段 → 强化社会证明（"4.6★ from 1,111+ users"）
- 减少直接价格展示（调研阶段用户价格敏感）
- 增加"See why X love it"类探索性CTA

**Category/Pain Points Campaign 文案**：
- 首次启动时沿用原广告方案的文案
- 根据投放数据后续迭代

### 4.5 预算分配建议

| Campaign | 日预算基准 | 调整规则 |
|----------|-----------|---------|
| Brand | 原30% | 有转化时维持，0转化时降至20% |
| Category | 原40% | 必须启动，优先分配 |
| Pain Points | 原30% | 优先启动高意图AG（如Dyson Alternative） |

## Step 6: 输出优化报告

详见 → `references/report-generator.md`

### 报告结构
1. 数据总览（关键指标汇总表）
2. 用户意图分析（意图分布、旅程映射、匹配度检查）
3. 核心问题诊断（P0/P1/P2分级）
4. 具体优化建议（否定词/关键词/出价/文案/预算）
5. 调整后的广告方案（可直接执行）
6. 优先级执行清单
7. 预期效果与风险提示

### 输出文件
- 优化报告：`brands/[BrandName]/[BrandName]-[ProductSlug]-Optimization-Report-v[X].md`
- 调整后广告：`brands/[BrandName]/[BrandName]-[ProductSlug]-Ready-To-Use-v[X].[Y].md`

---

## Step 7: 生成调整后的Ready-To-Use广告文件

> **用户确认检查点**: 在输出最终调整后的广告文件前，向用户展示关键调整摘要（暂停的关键词、新增的否定词、调整的出价等），确认后再生成最终文件。

### 展示给用户的关键调整摘要模板

```markdown
## 关键调整摘要

### 关键词调整
- 暂停: [N] 个（如 [keyword1], [keyword2]...）
- 新增: [N] 个（如 [keyword3], [keyword4]...）
- 匹配类型调整: [N] 个
- 出价调整: [N] 个

### 否定词调整
- 新增账户级否定词: [N] 个
- 新增Campaign级否定词: [N] 个

### Campaign/AG调整
- 启动: [N] 个未运行的广告组
- 出价策略调整: [N] 个Campaign

### 文案调整
- 调整标题: [N] 个
- 调整描述: [N] 个

**请确认以上调整后，我将生成最终的可执行广告文件。**
```

### 执行
1. 基于原广告方案（Step 0中读取的文件）
2. 应用Step 5中的所有调整
3. 输出可直接在Google Ads Editor中使用的格式
4. 版本号递增（如原v1.0 → 优化后v1.1）

### 格式要求
与google-ads-v6.3的Ready-To-Use格式完全一致：
- 关键词标注匹配类型（精确=[kw]、词组="kw"、广泛=+kw）
- 标题≤30字符，描述≤90字符
- 否定关键词用 - 前缀
- 包含完整广告扩展（Sitelinks/Callouts/Structured Snippets）

## 质量检查清单

### OPT-QC-1: 数据完整性
- [ ] 两个Excel报告都成功解析
- [ ] 总费用/点击/展示/转化数据已提取
- [ ] 搜索字词报告中的"已添加/已排除"状态已识别

### OPT-QC-2: 产品匹配性
- [ ] 搜索字词与广告方案品类一致
- [ ] 如有错配，已告知用户并使用正确文件

### OPT-QC-3: 问题诊断完整性
- [ ] 转化率问题已评估（0转化时标注为P0）
- [ ] 广告组结构覆盖率已检查（对比方案vs实际）
- [ ] 广泛匹配无关词已识别
- [ ] 零效果关键词已列出
- [ ] CPC异常（高于/低于均值）已识别

### OPT-QC-4: 优化建议可执行性
- [ ] 每条否定词都有具体来源（来自哪个搜索字词）
- [ ] 每条关键词调整都有数据依据
- [ ] 出价调整有具体百分比建议
- [ ] 新增关键词来自搜索字词报告中的高价值未添加词

### OPT-QC-5: 报告格式
- [ ] 包含优先级执行清单（今天/本周/两周内）
- [ ] 包含预期效果与风险提示
- [ ] 调整后广告可直接在Editor中使用

### OPT-QC-6: 广告文件合规
- [ ] 关键词匹配类型标注正确
- [ ] 标题≤30字符，描述≤90字符
- [ ] 否定关键词格式正确（-前缀）
- [ ] 包含完整的广告扩展
- [ ] 包含出价策略和预算

### OPT-QC-7: 用户意图分析完整性
- [ ] 搜索字词已按意图层级分类
- [ ] 意图分布统计已完成
- [ ] 典型用户旅程已还原
- [ ] 旅程断点已识别

### OPT-QC-8: 意图-广告匹配度
- [ ] 高费用搜索字词的意图已识别
- [ ] 当前触发的广告与意图匹配度已评估
- [ ] 意图错配案例已列出

### OPT-QC-9: 意图导向优化建议
- [ ] 每个意图层级有对应的文案优化建议
- [ ] 新增关键词建议按意图层级分类
- [ ] 否定关键词决策包含意图分析
