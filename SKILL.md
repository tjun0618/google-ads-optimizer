# Google Ads Optimization Skill — Amazon Affiliate Edition

> 版本: v2.1 | 2026-04-26
> 定位: 专为Amazon联盟营销客(Amazon Affiliate Marketer)设计的广告优化技能
> 上游依赖: google-ads-v6.3（广告创建）→ 本技能（投放优化）→ 持续迭代

---

## 核心差异：为什么Affiliate优化不同？

作为Amazon Affiliate，你的优化约束与普通电商完全不同：

| 约束 | 普通电商 | Amazon Affiliate |
|------|---------|-----------------|
| 落地页 | 可控，可A/B测试 | 不可控，直接到Amazon |
| 转化追踪 | Google Ads像素完整追踪 | 无法追踪，只能看Amazon Associates后台 |
| 出价策略 | Target CPA / Maximize Conversions 可用 | 只能用 Maximize Clicks 或人工出价 |
| 受众 | 可用RLSA再营销 | 无网站，无法收集受众 |
| 利润模型 | 销售收入 - 广告成本 | Amazon佣金 - Google Ads支出 |
| Cookie窗口 | 无限制（自己的转化） | 24小时Amazon Associates Cookie |

**Affiliate优化的核心公式**（详见 → `references/affiliate-profit-analyzer.md`）:

```
盈亏平衡CPC = 客单价 × 佣金率 × Amazon预估转化率
安全CPC上限 = 盈亏平衡CPC × 安全系数(0.3-0.7)
```

所有优化动作必须围绕**降低无效点击成本**和**提高流量购买意图**展开。

> **为什么意图分析先于利润分析？**
> 意图分析回答"流量质量如何"（哪些搜索有购买意图、哪些是无效浏览），这是利润分析的输入。
> 利润分析回答"这些流量值多少钱"（基于意图质量计算盈亏平衡CPC）。
> 没有意图分析，利润分析就是无的放矢。

## Step 0: 接收文件并验证匹配性

### 输入
- 用户提供的3-4个文件路径

### 执行
1. 确认必要文件（1-3）都存在且可读
2. 如有Amazon Associates订单报告（文件4），一并读取
3. 读取广告方案文件中的产品名称、ASIN、价格、佣金率
4. 读取Excel报告中的关键词/字词样本，判断是否与广告方案的产品匹配

### 匹配性判断规则
- 搜索关键字报告中的关键词应与广告方案中的产品品类一致
- 例如：广告方案是吹风机(hair dryer)，报告中应包含hair dryer相关词
- 如果完全不匹配（如广告是喂猫机但报告全是猫砂盆词），需立即告知用户并确认

### 输出
- 产品信息确认摘要（含佣金率、盈亏平衡CPC）
- 匹配性判断结果（匹配/不匹配/需确认）

### 边界条件处理

**文件不存在或不可读**:
```
如果任一必要文件（1-3）不存在:
  1. 列出所有缺失的文件路径
  2. 告知用户需要提供的文件
  3. 停止执行，等待用户重新提供
```

**数据量不足（<10条有效记录）**:
```
如果搜索关键字报告的有效关键词<10个:
  1. 告知用户数据量可能不足以进行有效分析
  2. 建议延长投放时间或增加预算以积累更多数据
  3. 继续执行分析，但在报告中标注"数据量有限，建议谨慎采纳"
```

**两个报告时间范围不一致**:
```
如果搜索关键字报告和搜索字词报告的时间范围不同:
  1. 提取两个报告的时间范围
  2. 告知用户时间范围不一致可能影响分析准确性
  3. 建议重新导出同一时间范围的报告
  4. 继续执行分析，但在报告中标注时间范围差异
```

**Amazon Associates订单报告格式错误**:
```
如果用户提供了Amazon Associates订单报告但无法解析:
  1. 尝试识别文件格式（CSV/TSV/XLSX）
  2. 检查是否包含必需的列（Date/ASIN/Commission）
  3. 如果缺少关键列:
     - 告知用户报告格式不匹配
     - 说明需要的列名和格式
     - 标记为"无Amazon订单数据"，继续用预估模型分析
  4. 如果解析成功但无目标ASIN的订单:
     - 告知用户未检测到该产品订单
     - 检查是否 Associates Tag 正确
     - 继续分析，但标注"实际EPC=0（可能追踪遗漏或确实无转化）"
```

**Amazon产品不可用或库存紧张**:
```
在Step 0中主动检查（详见 → references/amazon-affiliate-checklist.md）:
  1. 读取广告方案中的ASIN
  2. 检查Amazon产品页面状态
  3. 如果 Currently unavailable / Only X left (X<10):
     - 标记为P0风险
     - 建议立即暂停Campaign
     - 在利润分析中将预估转化率设为0
```

- 搜索字词报告：字词列表、匹配类型、广告组、已添加/已排除状态、展示/点击/费用

## Step 2: 产品-报告匹配性验证

### 执行
1. 从搜索字词报告中提取出现频率最高的核心词（排除停用词后）
2. 与广告方案文件中的产品品类关键词对比
3. 如果搜索字词与广告方案的品类完全不同，标记为"产品错配"

### 产品错配处理
- 告知用户：报告中数据对应的是另一个产品
- 查找该产品对应的广告方案文件
- 经用户确认后，使用正确的广告方案文件继续优化

---

## Step 3: 用户意图分析

详见 → `references/user-intent-analyzer.md`

### 核心输出
- 搜索字词按意图层级分类（购买决策/产品调研/方案探索/问题认知/信息浏览）
- 意图分布统计（各层级的字词数、点击、费用）
- 典型用户旅程映射（从搜索词还原用户的决策路径）
- 旅程断点识别（哪些阶段无广告覆盖）
- 意图-广告匹配度检查（当前广告是否回应了用户的真实意图）
- 基于意图的文案优化建议

### 意图分析如何驱动利润分析

**发现1: 意图分布失衡**
- 如果90%点击来自"信息浏览"意图 → 流量质量差 → 利润分析时应收紧CPC上限
- 如果"方案探索"意图完全缺失 → Category Campaign未启动 → 利润分析时标注高利润机会

**发现2: 旅程断点**
- 用户从"tired of heavy dryer"（Aware）开始 → 但无Pain Points Campaign → 断点=利润损失

**发现3: 意图-广告错配**
- 用户搜"reviews"但看到硬卖广告 → 调研意图错配 → 影响该流量段的实际转化率

## Step 4: Affiliate利润分析 ★核心

详见 → `references/affiliate-profit-analyzer.md`

### 前置输入
- Step 3的意图分析结果（意图分布、旅程断点、意图-广告匹配度）
- 广告方案中的产品信息（售价、佣金率、ASIN）
- 解析后的Google Ads数据（关键词CPC、点击量）
- 可选：Amazon Associates订单报告

### 核心输出
- **盈亏平衡CPC**：基于产品佣金率、预估Amazon转化率计算
- **关键词利润评级**：每个关键词/广告组的预期利润（A高利润/B盈利/C边缘/D亏损）
- **实际EPC估算**：如有Amazon订单报告，计算实际每点击收益
- **亏损词识别**：CPC超过盈亏平衡点的关键词
- **高利润机会**：低CPC + 高购买意图的关键词（通常来自Explore/Aware意图）

### 利润分析如何驱动优化

**发现1: CPC超过盈亏平衡**
- 某关键词CPC $2.00，但盈亏平衡CPC仅 $1.50 → 每点击亏损$0.50 → 立即暂停或降价

**发现2: 高利润Campaign未启动**
- Category Campaign（方案探索意图）未启动 → 低CPC + 高佣金 = Affiliate金矿 → 必须启动

**发现3: 实际EPC与预估差距**
- 预估EPC $1.80，实际EPC $0.30 → Amazon页面转化率远低于预期 → 检查产品评论/库存/价格

### 用户确认检查点 ★新增

> **利润模型确认**: 完成利润分析后，向用户展示盈亏平衡CPC和安全上限，确认后再继续诊断。

```markdown

| 产品 | [产品名] | $[价格] | [评分]★ |

| 预估Amazon转化率 | [X]% | 品类基准 |
| **盈亏平衡CPC** | **$[X]** | 核心红线：CPC必须低于此值 |
```

### 6.1 政策合规与库存检查（★新增）

在生成具体优化方案前，先执行Affiliate专用合规检查（详见 → `references/amazon-affiliate-checklist.md`）：

**Amazon Associates合规**:
- [ ] 广告文案中的价格是否与Amazon一致（或标注"Price may vary"）
- [ ] 是否包含Amazon Associates免责声明（如使用内容桥接）
- [ ] 是否存在 incentivize 点击的措辞（如"click my link for discount"）

**Google Ads政策合规**:
- [ ] 落地页是否为Amazon实际产品页（非Bridge Page）
- [ ] 广告文案是否存在夸大声明（特别是健康/保健品类）
- [ ] 是否存在虚假折扣/优惠声明

**库存与价格风险**:
- [ ] Amazon产品当前是否可用
- [ ] 价格是否有大幅波动（>20%）
- [ ] 评分是否有下降（<4.0或近期大量负面评价）

如有任何合规或库存风险，在报告中标注并建议立即处理。

---

### 6.2 否定关键词调整

根据搜索字词报告，生成新增否定词列表：

**账户级否定词新增规则**：
- 竞品品牌名（搜索字词中出现但非己方品牌的品牌词）
- 完全无关品类词（如广告是吹风机但触发了bonnet hair dryer、diffuser等）
- 不同产品类型词（如reverse hair dryer、stick blow dryer）
- 非目标语言词（如西班牙语、法语等美国市场非英语搜索）
- 教程/信息类搜索（tutorial、how to、without rollers等）
- 极低意图泛词（amazon hair dryer、hair dryer heat等）
- **Affiliate专用**："free"、"giveaway"、"sample"、"walmart"、"target"、"refurbished"、"used"

**Campaign级否定词新增规则**：
- Brand Campaign：排除社区讨论平台（-reddit、-quora、-forum）、排除品牌旗下其他产品线
- Category Campaign：排除二手/维修/配件类
- Pain Points Campaign：保留竞品对比词（vs、compare、alternative），这些在痛点Campaign中有价值

### 6.3 关键词调整（基于利润分析）

详见 → `references/optimization-analyzer.md` 中的关键词调整规则。

核心原则：
- **亏损词**：CPC > 盈亏平衡CPC → 暂停
- **高利润词**：CPC < 安全上限 × 0.5 → 加码（提高出价10-20%）
- **广泛匹配失控**：无关触发≥5条 → 收紧匹配类型
- **零效果词**：展示>200且点击=0 → 暂停

### 6.4 出价策略调整（Affiliate专用）

详见 → `references/optimization-analyzer.md` 中的出价策略规则。

核心原则（所有Campaign一律）：
- **策略**：Maximize Clicks（Affiliate无法追踪转化，不能用Maximize Conversions）
- **Brand Campaign上限**：盈亏平衡CPC × 0.7
- **Category Campaign上限**：盈亏平衡CPC × 0.6
- **Pain Points Campaign上限**：盈亏平衡CPC × 0.5

### 6.5 文案调整原则

详见 → `references/user-intent-analyzer.md` 中的意图导向文案框架。

核心原则：
- **调研意图**（reviews/reddit）→ 社会证明型文案（评分、评论数）
- **探索意图**（best/top）→ 差异化卖点型文案
- **痛点意图**（tired of/fix）→ 痛点共鸣型文案
- **Affiliate注意**：CTA必须导向Amazon，文案与产品页一致

### 6.6 预算分配建议

详见 → `references/affiliate-profit-analyzer.md` 中的高利润机会识别。

核心原则：
- Category Campaign利润效率通常最高 → 优先启动和分配预算
- Brand Campaign CPC较高 → 有订单支撑时维持，无订单时降预算
- Pain Points Campaign转化链路长 → 保守预算

## Step 8: 生成调整后的Ready-To-Use广告文件

### 展示给用户的关键调整摘要模板

```markdown

### 利润影响
```
